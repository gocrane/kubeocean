# Tapestry 项目规则

本目录包含 Tapestry 项目的核心开发规则和指南。

## 文档结构

### [core-rules.md](./core-rules.md) - 核心规则速查
最简洁的核心规则，包含：
- 代码规范要点
- 架构设计原则  
- 测试要求
- 资源规范
- 关键命令

**适用场景**: 快速查阅、新人入门、代码审查参考

### [project-rules.md](./project-rules.md) - 完整项目规则
详细的项目规范文档，包含：
- 代码风格和命名规范
- 架构设计原则和模式
- 测试策略和规范
- 资源管理规范
- 错误处理和日志规范
- 构建部署规范
- 安全和监控要求
- 版本管理规范

**适用场景**: 深入了解项目规范、架构设计参考、规范制定

### [development-guide.md](./development-guide.md) - 开发指南
实用的开发指南，包含：
- 快速开始步骤
- 编码规范速查
- 测试编写模式
- 常用代码片段
- 调试技巧
- 提交规范

**适用场景**: 日常开发参考、代码模板、问题排查

## 使用建议

1. **新开发者**: 先阅读 `core-rules.md` 了解基本规范，然后查看 `development-guide.md` 开始开发
2. **日常开发**: 使用 `development-guide.md` 作为快速参考和代码模板
3. **架构设计**: 参考 `project-rules.md` 了解完整的设计原则和规范
4. **代码审查**: 使用 `core-rules.md` 作为检查清单

## 规则更新

这些规则基于项目当前状态制定，包含：
- 现有代码风格和架构模式
- `.golangci.yml` 配置
- `Makefile` 构建规范  
- 文档和示例中的最佳实践
- 实际开发过程中的编码习惯

规则会随着项目发展持续更新，建议定期查阅最新版本。
